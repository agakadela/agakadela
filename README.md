The most expensive software mistakes don’t throw errors.
You notice them when the revenue, data, or user trust is already wrong.

I spent 1.5 years building checkout systems for one of Europe’s largest e-commerce platforms — 1M+ users, thousands of daily transactions, and code paths where a small bug could stop money from moving.

That shaped how I work now.

I build and harden AI-powered SaaS products that need to survive real users, payments, private data, and production traffic.

Most of my work sits around:
- AI products that need more than a demo
- SaaS platforms with auth, billing, roles, and data boundaries
- Stripe/payment flows where access must match reality
- background jobs, webhooks, and async workflows that cannot silently fail
- AI endpoints with cost controls, usage limits, and observability

I care about the boring parts that become expensive later:
authorization, multi-tenancy, billing state, idempotency, rate limits, error handling, monitoring, and recovery.
